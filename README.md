# Von Frey Up-Down Analysis Tool

A GUI and CLI tool for computing 50% withdrawal thresholds from von Frey up-down data and producing publication-ready figures with statistical analysis.

## Background

The von Frey test is a standard method for assessing mechanical sensitivity in rodents. An animal is placed on a mesh platform and its hindpaw is probed with calibrated nylon monofilaments of increasing or decreasing force. The experimenter records whether the animal withdraws its paw (positive response, `x`) or not (negative response, `o`).

The **up-down method** (Dixon, 1980; Chaplan et al., 1994) is an efficient procedure that adjusts the stimulus intensity based on the animal's response: after a withdrawal, a lighter filament is applied (step down); after no withdrawal, a heavier filament is applied (step up). The sequence of responses is recorded as a string of `x` and `o` characters (e.g., `oxooxo`), and this pattern, together with the final filament used, determines the **50% withdrawal threshold** — the estimated force at which the animal has a 50% probability of withdrawing.

This tool automates the threshold computation from raw response series, generates publication-quality figures for longitudinal and factorial experimental designs, and provides built-in statistical analysis with multiple comparison correction.

## Features

- **50% threshold computation** using the Dixon up-down method with tabulated k-statistics
- **Two experimental designs supported:**
  - **Longitudinal** (multiple timepoints, 2 groups) — individual animal traces + group mean line plots
  - **Factorial / pre-post** (few timepoints, multiple groups) — paired line plots and delta (change score) plots
- **Statistical analysis:** repeated-measures ANOVA, pairwise t-tests, mixed-effects models, delta score ANOVA
- **Multiple comparison correction:** Holm-Bonferroni, Bonferroni, Benjamini-Hochberg (FDR)
- **Publication-ready figures:** Arial font, editable PDF output (type 42 fonts for Adobe Illustrator), log-scale y-axis, sex-specific markers/linestyles, despined axes
- **Export:** PDF, PNG, SVG figures; Excel/CSV data and statistics tables
- **GUI** with live plot preview and step-by-step workflow
- **CLI** mode for batch threshold computation
- **Session save/load** to preserve your analysis configuration

## The Dixon Up-Down Method

The 50% withdrawal threshold is computed using the formula:

```
threshold = 10^(Xf + k * d) / 10,000
```

Where:

| Variable | Meaning |
|----------|---------|
| **Xf** | Log value of the final filament in the series |
| **k** | Tabulated statistic determined by the x/o response pattern |
| **d** (delta) | Mean log interval between filaments = 0.4414 |

The log value of each filament is computed from its force as: `Log = log10(10 * force_in_grams * 1000)`.

### Reference file: `VF_Calculator_Up-down.xlsx`

This file ships with the repository in the `data/` folder and contains two lookup tables:

1. **Filament reference table** — Calibration data for 8 von Frey filaments:

   | Filament | Force (g) | Log | Marking |
   |----------|-----------|-----|---------|
   | 1 | 0.0045 | 1.65 | 0.008 |
   | 2 | 0.0230 | 2.36 | 0.020 |
   | 3 | 0.0680 | 2.83 | 0.070 |
   | 4 | 0.1580 | 3.22 | 0.160 |
   | 5 | 0.1780 | 3.61 | 0.400 |
   | 6 | 1.2020 | 4.08 | 1.000 |
   | 7 | 2.0410 | 4.31 | 2.000 |
   | 8 | 5.4950 | 4.74 | 6.000 |

2. **k-statistic lookup table** — 248 entries mapping every possible x/o response pattern (2 to 9 characters) to its corresponding k value. For example: `OX → -0.500`, `OXOOXO → 0.168`, `OOXXOO → 0.000`.

> **Do not edit this file.** It contains the standard calibration values required for accurate threshold computation.

### Log column choice

The tool computes its own log values (`Log_new`) from the force data, which may differ slightly from the `Log` column in the reference file due to log value of filament 4. By default, `Log_new` (computed values) is used. You can switch to the original `Log` column in the GUI or via the `--log-column` CLI flag.

## Data Format Requirements

### Experimental data file

An Excel (`.xlsx`) or CSV file with one row per mouse per timepoint. Required columns:

| Column | Description | Example |
|--------|-------------|---------|
| `mouse` | Unique animal identifier | `1441` |
| *timepoint column* | Timepoint label (any column name) — numeric (days) or categorical (text) | `-1`, `3`, `14` or `pre`, `post_chronic` |
| `xo_series` | String of `x` (withdraw) and `o` (no withdraw) characters | `oxooxo` |
| `last_filament` | Integer number (1-8) of the final filament in the series | `5` |

The timepoint column can have any name. You map it in the GUI or specify the design in the metadata.

### Metadata file (optional)

An Excel or CSV file with one row per mouse. Used to assign groups, sex, and other experimental variables.

| Column | Description | Example |
|--------|-------------|---------|
| `mouse` | Must match the data file | `1441` |
| `gender` | `male` or `female` — used for sex-specific markers and linestyles | `female` |
| *group column(s)* | Any column(s) defining experimental groups | `group`, `group_exp`, `phase` |

## GUI Workflow

The GUI follows a 6-step workflow accessible from the left sidebar.

### Step 1: Data

- Load your **experimental data file** (Excel or CSV)
- Load the **filament reference file** (`VF_Calculator_Up-down.xlsx`, included in `data/`)
- Optionally load a **metadata file** for group/sex information
- Map your column names (mouse, timepoint, xo_series, last_filament)
- Choose the log column (`Log_new` recommended)
- Click **Compute Thresholds** — the tool calculates 50% thresholds for every row

### Step 2: Groups

- Select which column(s) define your experimental groups
- Assign mice to groups (auto-detected from metadata if provided)
- Exclude or reorder timepoints as needed
- Set an **intervention marker** (optional) — a vertical dashed line at any x-value (e.g., day 0 for surgery), not restricted to existing timepoints

### Step 3: Appearance

- Select **plot type**: longitudinal line plot, paired line plot, or delta plot
- Set colors for each group
- Configure markers (circle = male, triangle = female) and line styles (solid = male, dotted = female)
- Adjust axis labels, title, and figure size
- Configure y-axis log scale and x-axis spacing

### Step 4: Preview

- Live matplotlib canvas showing your figure
- Updates as you change settings in other panels
- Longitudinal plots show individual animal traces (thin, semi-transparent) overlaid with group mean ± SEM (thick lines)

### Step 5: Statistics

- Select a **statistical test**:
  - Repeated-measures two-way ANOVA (default for longitudinal data)
  - Pairwise t-tests at each timepoint (with correction across all comparisons)
  - Linear mixed-effects model (for missing data or unbalanced designs)
  - Delta score ANOVA (for pre-post designs)
- Select a **multiple comparison correction**: Holm-Bonferroni (default), Bonferroni, or Benjamini-Hochberg (FDR)
- Run the analysis — results appear as a summary table and as significance annotations on the plot

### Step 6: Export

- Save figures as **PDF** (primary, with editable fonts for Illustrator), **PNG**, or **SVG**
- Export threshold data and statistical results as **Excel** or **CSV**
- Save/load analysis **sessions** (JSON) to resume work later (also via File menu, Ctrl+S / Ctrl+O)

## Command-Line Interface

Use `--compute` mode to calculate thresholds without the GUI:

```bash
python run.py --compute \
  --data data.xlsx \
  --metadata info_mice.xlsx \
  --filament-ref data/VF_Calculator_Up-down.xlsx \
  --output results/
```

| Argument | Description | Default |
|----------|-------------|---------|
| `--compute` | Run in CLI mode (without GUI) | — |
| `--data` | Path to von Frey data file (required in compute mode) | — |
| `--metadata` | Path to metadata file (optional) | — |
| `--filament-ref` | Path to filament reference file | `data/VF_Calculator_Up-down.xlsx` |
| `--output` | Output directory for results | `.` (current directory) |
| `--log-column` | Log column to use: `Log_new` or `Log` | `Log_new` |
| `--gui` | Launch the GUI (default when no flags given) | `true` |

The CLI outputs a file `vf_thresholds.xlsx` in the specified output directory, containing your original data with a `threshold_50` column appended.

## Statistical Methods

### Repeated-Measures Two-Way ANOVA

The default test for longitudinal designs with two or more groups and multiple timepoints. Uses `pingouin.mixed_anova` with the group as a between-subject factor and timepoint as a within-subject factor.

- Tests for main effects of group, timepoint, and their interaction
- Applies **Greenhouse-Geisser correction** automatically when sphericity is violated (>2 timepoints)
- Reports: F-statistic, degrees of freedom, p-value, partial eta-squared (effect size)
- Requires complete cases (subjects with all timepoints); incomplete subjects are dropped with a warning

### Pairwise t-Tests with Multiple Comparison Correction

Compares groups at each timepoint individually using **Welch's t-test** (does not assume equal variances). All p-values across all timepoints are collected and corrected simultaneously.

- Reports per comparison: t-statistic, Welch-Satterthwaite df, uncorrected p, corrected p, Cohen's d
- Simpler to interpret than ANOVA when the question is "at which timepoints do the groups differ?"

### Linear Mixed-Effects Model

For advanced users or when data has missing values / unbalanced groups. Fits `threshold ~ C(group) * C(timepoint)` with a random intercept for each mouse.

- Timepoint is treated as categorical by default (no linearity assumption)
- Falls back to an additive model (no interaction) if the full model fails to converge
- Reports fixed-effects coefficients, standard errors, z-values, and p-values

### Delta Score Analysis (Pre-Post Designs)

For factorial designs with a clear pre and post timepoint:

1. Computes **delta scores** (post - pre) for each animal
2. Runs **ANOVA** on the delta scores with group/condition factors
3. Post-hoc pairwise comparisons on delta scores with correction

### Multiple Comparison Correction Methods

| Method | Description |
|--------|-------------|
| **Holm-Bonferroni** (default) | Sequentially rejective procedure; more powerful than Bonferroni while controlling family-wise error rate |
| **Bonferroni** | Divides alpha by the number of comparisons; most conservative |
| **Benjamini-Hochberg (FDR)** | Controls false discovery rate; more powerful, suitable when many comparisons are made |

### Significance Annotations on Plots

Corrected p-values are displayed on the plot as:
- `*` p < 0.05
- `**` p < 0.01
- `***` p < 0.001
- `n.s.` not significant

Exact corrected p-values are shown below the stars.

## Troubleshooting

### NaN threshold values

A threshold is `NaN` when the x/o series pattern is not found in the k-statistic lookup table. Common causes:
- The series contains characters other than `x` and `o`
- The series is empty or the cell is blank
- The `last_filament` value is outside the range 1-8

Check the "NaN" count reported after threshold computation and inspect the corresponding rows in your data.

### Mixed-effects model fails to converge

This typically occurs when:
- A group has only 1 subject (the random intercept cannot be estimated)
- Some group-by-timepoint cells are empty or have only 1 observation
- There is zero variance within a cell

The tool will attempt a simpler additive model (no interaction term) as a fallback. If both models fail, consider using repeated-measures ANOVA or pairwise t-tests instead.

### Missing timepoints / incomplete subjects

- **RM-ANOVA** requires complete cases and will drop subjects with missing timepoints (a warning is shown)
- **Pairwise t-tests** handle missing data naturally — each timepoint is analyzed independently
- **Mixed-effects models** can accommodate missing data without dropping subjects

### Figures not editable in Illustrator

The tool exports PDFs with `fonttype=42` (TrueType outlines) so text remains editable in Adobe Illustrator. If fonts appear as outlines, verify that Arial is installed on your system.
